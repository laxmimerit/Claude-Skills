# Hunyuan3D → Blender Skill

## Pipeline Order

1. `server_health` — stop if unreachable
2. `generate_3d_async` — kick off generation immediately, one object at a time if multiple
3. Build scene while waiting — base disc + lights + camera only
4. `check_generation_status` — poll until complete
5. Convert path if on Windows
6. Import, scale, seat, shade smooth
7. Apply color and texture in Blender with mandatory verification
8. Final verification against user request
9. Screenshot to confirm

---

## Critical Rules

- **The HY3D mesh is the only hero object.** Never build a procedural version of the requested model while waiting — not even as a placeholder.
- **One hero per scene** unless the user explicitly requests multiple.
- **Never auto-clear the scene.** If the scene needs clearing, ask the user first.
- **Never guess scale or position.** Always measure bounds after import and derive placement from real numbers — no hardcoded offsets ever.
- **Never use raw Linux paths in Blender on Windows.** Convert first — see Path Conversion below.

---

## Hunyuan3D is the Only 3D Modeler — CRITICAL

**Every 3D object in the scene must be generated by Hunyuan3D MCP tools. No exceptions.**

- If the design requires multiple 3D objects, generate each one separately using Hunyuan3D — one `generate_3d_async` call per object.
- Never model any 3D object directly in Blender using primitives, curves, or any Blender geometry tools. Blender's role is strictly: import, place, color, light, and camera.
- The user's description of the scene (e.g. "island with rockets and trees") is a list of objects to generate via Hunyuan3D — not a Blender modeling instruction.

**Blender is only permitted to:**
- Import HY3D meshes
- Place and seat them using real measured coordinates
- Apply colors and materials
- Set up lights and camera
- Hold the base display disc

---

## Strict Object Restriction — CRITICAL

**After HY3D meshes are imported, NO additional Blender geometry may be created unless the user has explicitly and specifically requested it by name.**

- The HY3D mesh defines the look of the scene. Direct Blender modeling on top of it will destroy the aesthetic of the AI-generated mesh.
- Do not add any geometry based on assumptions or inferred intent from the user's description.
- Only act on what the user explicitly says to add as a separate Blender object, stated clearly and directly.

**The ONLY exception** is the base display disc (see While Waiting below). Everything else requires an explicit user instruction.

---

## Hunyuan3D Prompt Rules — Mesh First, Color in Blender

**Always generate meshes without color or texture from Hunyuan3D.**

- Keep Hunyuan3D prompts geometry-focused: describe the shape, form, and structure only.
- Do not include color, material, or texture descriptions in the Hunyuan3D prompt — these will be applied in Blender after import.
- Avoid open-ended style words ("fantasy", "magical", "tropical") that cause Hunyuan3D to hallucinate extra geometry like trees, vegetation, or decorations.
- Keep prompts tight and specific to the object's physical form only.

**Good prompt:** `"a chunky floating rock island, rounded bottom, flat top surface, no vegetation, no trees"`
**Bad prompt:** `"a magical fantasy floating island with grass and palm trees"`

All color, texture, and material decisions happen in Blender — not in Hunyuan3D.

## While Waiting — Scene Setup

Build exactly three things, nothing else: a base disc, a 3-point lighting rig, and a camera set as `scene.camera`.

**Base:** bevelled cylinder, radius ~1.4, neutral/stone material, shade smooth. This is a scene display prop — it is the platform the HY3D mesh sits on. It is the only object permitted without explicit user instruction.

**Lighting:** 3 area lights — warm key from front-right, cool fill from front-left, accent rim from behind. Tune color and energy to match the subject's mood.

**Camera:** 85mm lens, 3/4 hero angle looking down at the model. Set as `scene.camera`.

## Path Conversion — Windows + WSL

HY3D runs in WSL and returns a Linux path. Blender on Windows cannot read it directly. Always convert before importing:

```python
import sys

def linux_to_windows_wsl(p):
    return "\\\\wsl.localhost\\Ubuntu" + p.replace("/", "\\")

glb_path = linux_to_windows_wsl(linux_path) if sys.platform == "win32" else linux_path
```

If the WSL distro is not `Ubuntu`, replace with the correct name (e.g. `Ubuntu-22.04`).

## Import & Seat Rules

- Deselect using `obj.select_set(False)` on all objects — never `bpy.ops.object.select_all` which crashes outside viewport context.
- After import, collect only `MESH` type objects from `bpy.context.selected_objects`.
- If the result is empty, stop and raise a clear error — do not proceed silently.
- Rename all imported objects with `HY3D_` prefix.
- Scale uniformly so the model fits `target_height=2.0` units. Derive scale factor from measured Z bounds, never guess.
- Re-measure bounds after scaling. Centre on X/Y. Seat bottom at `z=0` (top of base disc).
- Shade smooth after seating — use `poly.use_smooth = True` on `obj.data.polygons` directly, never `bpy.ops.object.shade_smooth` which requires viewport context.
- After seating, check model orientation. If it faces away from camera, rotate on Z axis to face front.
- Print final height, bottom_z, and scale factor to confirm correctness.

### Multi-Object Placement Rules

When the scene has more than one HY3D object:

- Measure the real bounding box of every object after import and scaling.
- Calculate exact placement coordinates from those measured numbers so objects do not overlap, float, or interpenetrate.
- No hardcoded offsets — every X, Y, Z position must be derived from actual bounds arithmetic.
- Print final position of every object to stdout before proceeding.

## Material & Color Rules

- All coloring and texturing happens in Blender after import — never in the Hunyuan3D prompt.
- Always clear all existing GLB baked materials from the object first before assigning anything new.
- Assign a fresh Principled BSDF for every object.
- Match base color and roughness to the user's specified values exactly (hex or described color).
- Never leave the GLB's original baked materials in place — they will render white or incorrect in Cycles.

### Mandatory Material Verification

After assigning every material, run the following checks before proceeding:

```python
# 1. Confirm material is assigned
assert obj.data.materials[0].name == mat.name, "Material not assigned!"

# 2. Confirm Principled BSDF exists and is connected to Material Output
nodes = mat.node_tree.nodes
links = mat.node_tree.links
bsdf = nodes.get("Principled BSDF")
assert bsdf is not None, "Principled BSDF node missing!"
output = nodes.get("Material Output")
assert output is not None, "Material Output node missing!"
connected = any(
    l.from_node == bsdf and l.to_node == output
    for l in links
)
assert connected, "Principled BSDF not connected to Material Output!"

# 3. Print actual color to stdout for confirmation
color = bsdf.inputs["Base Color"].default_value
print(f"Material '{mat.name}' verified — color: ({color[0]:.3f}, {color[1]:.3f}, {color[2]:.3f})")
```

If any check fails — re-assign the material from scratch and re-verify. Never proceed with an unverified material.

## Final Verification Against User Request — MANDATORY

Before taking the final screenshot, explicitly verify the scene against the user's original request:

- **Object count:** Does the number of HY3D objects match what the user asked for?
- **Object types:** Does each object correspond to something the user specifically requested?
- **No extra objects:** Are there any objects in the scene the user did not ask for? If yes — delete them before proceeding.
- **Colors:** Does each object's verified material color match the user's specified color?
- **Placement:** Is every object positioned correctly — seated, not floating, not overlapping?
- **Scene cleanliness:** Only permitted objects exist — HY3D meshes, base disc, lights, camera.

Print a verification summary to stdout listing each check as PASS or FAIL:

```python
print("=== FINAL VERIFICATION ===")
print(f"Object count match:   PASS/FAIL")
print(f"No extra objects:     PASS/FAIL")
print(f"Colors verified:      PASS/FAIL")
print(f"Placement verified:   PASS/FAIL")
print(f"Scene clean:          PASS/FAIL")
print("==========================")
```

If any check is FAIL — fix it before taking the screenshot. Do not skip this step.

## Troubleshooting

| Symptom | Fix |
|---|---|
| `RuntimeError: Please select a file` | Missing path conversion — run `linux_to_windows_wsl()` |
| `imported = []` | Wrong WSL distro name, or file not yet written |
| `poll() failed, context is incorrect` | Replace `bpy.ops.object.select_all` with `obj.select_set(False)` loop; use `poly.use_smooth` instead of `bpy.ops.object.shade_smooth` |
| Model too large or tiny | Re-derive scale factor from actual Z bounds |
| Model faces wrong way | Rotate on Z axis after import |
| Floating mesh fragments | Edit Mode → Merge by Distance |
| Material renders white in Cycles | GLB baked material overriding — clear all materials first, then assign fresh Principled BSDF, then verify |
| Material verification assertion fails | Re-create material node tree from scratch — do not patch existing nodes |
| Objects overlapping or floating | Re-measure all bounds, recalculate positions from real numbers — never guess |
| Hunyuan3D generated unwanted geometry | Prompt was too open-ended — rewrite prompt to be geometry-specific only and re-generate |
